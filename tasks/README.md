# Task memory

Material changes get one scratchpad named `YYYY-MM-DD-HHMM-slug.md`. Keep it concise and update it
in place so later sessions can resume without replaying exploration.

Required fields:

- `Status:` active, blocked, or complete
- Goal and non-goals
- Evidence and affected files
- Decisions/assumptions
- Blast radius and validation plan
- Final diff summary, checks run, results, risks, and deferred work

Do not store credentials, full command logs, copied source, or transient reasoning. Completed
scratchpads are durable decision records; prune only when their useful facts have moved to an ADR or
maintenance document.
